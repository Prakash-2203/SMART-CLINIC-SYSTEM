1. View Patient Records

User Story: As a doctor, I can view the patient record so that I can provide accurate medication and treatment.

Priority: High

Story Points: 5

Acceptance Criteria:

Doctor can search/select a patient profile.

System displays medical history, previous prescriptions, and visit notes.

Only authorized doctors can access patient details.

2. View Scheduled Appointments

User Story: As a doctor, I can see my scheduled appointments so that I can manage my day effectively.

Priority: High

Story Points: 3

Acceptance Criteria:

Doctor dashboard shows appointment list for current/future dates.

Every appointment includes patient name, time, and reason for visit.

Doctor can filter appointments by day/week.

3. Add Prescription Notes

User Story: As a doctor, I can add prescription notes to an appointment so that the pharmacy can fulfill the medication.

Priority: Medium

Story Points: 4

Acceptance Criteria:

Doctor can enter/edit prescription notes for each visit.

Notes automatically save to the patient record.

Pharmacy and patient can view the prescription after submission.

üßë‚Äçü¶∞ Patient User Stories
1. View Upcoming Appointments

User Story: As a patient, I can view my upcoming appointments so that I can prepare for consultations.

Priority: High

Story Points: 3

Acceptance Criteria:

Patient dashboard displays future booked appointments.

Each appointment shows doctor name, specialty, date, and time.

Patient receives confirmation when an appointment is updated.

2. Search for Doctors

User Story: As a patient, I can search for doctors by name or specialty so that I can book the right appointment.

Priority: High

Story Points: 5

Acceptance Criteria:

Search bar allows filtering by name and specialty.

System returns a list of matching doctors with availability.

Patient can open doctor profiles before booking.

3. Cancel Appointments

User Story: As a patient, I can cancel my booking so that I can manage changes in my schedule.

Priority: Medium

Story Points: 2

Acceptance Criteria:

Patient can cancel from appointment details page.

System sends a cancellation confirmation.

Time slot becomes available for others to book.

1. Add or Remove Doctors

User Story: As an admin, I can add or remove doctors so that the clinic staff list remains up to date.

Priority: High

Story Points: 5

Acceptance Criteria:

Admin can add doctor details (name, specialty, availability).

Admin can deactivate/remove doctor profiles.

System validates required fields before saving.

2. View Appointment Reports

User Story: As an admin, I can view system reports of appointments so that I can track clinic utilization.

Priority: Medium

Story Points: 8

Acceptance Criteria:

Admin can view daily, weekly, and monthly appointment reports.

Reports show total appointments per doctor and per date.

Admin can download reports (CSV/PDF optional).

3. Manage User Accounts

User Story: As an admin, I can assign roles and manage user accounts so that access is properly controlled.

Priority: High

Story Points: 8

Acceptance Criteria:

Admin can create, update, or delete user accounts.

Admin can assign roles: Doctor, Patient, Admin.

System restricts functionality based on assigned role.
